is-valid-BST

发表于 2021-04-08

预备知识

  1. 二叉搜索树是指
    1. 每个节点中的值必须大于（或等于）其左子树中的任何值。
    2. 每个节点中的值必须大于（或小于）其左子树中的任何值。
  2. 二叉搜索树的中序遍历是有序的（从小到大）

1. 暴力解法

先中序遍历得到“有序”序列，但检测序列是否真的有序。该想法直截了当，直接基于BST的特殊性质。但使用了额外的内存空间，对于有n个节点的二叉树，空间复杂度为O(n)，最坏需要遍历两次所有节点值。

2. 尝试提高效率

当要解决一个问题时，常常需要分步骤解决。就如暴力解法中一般，我们先拿到序列，然后再遍历。如果我们向压缩空间，那就不允许使用额外的内存空间来记录所有的中间值。这个时候，我们就可能会思考，我们是否可以再第一次遍历的适合就尝试进行比较呢？就像插入排序一般。
最简单的方法就是利用全局变量来记录参考值。此时空间复杂度降低到O(1)。
这里可以看出，当如果有个变量在每次递归中都需要使用到，不要犹豫，直接使用全局变量。不过，在实际过程，需要防止方法污染全局变量。

  • 左节点: 必须小于父节点，故 $val \subseteq (MIN, parent)$

二叉树中的最大路径

发表于 2021-04-06 更新于 2021-04-07

分析过程

叶节点

对于一个空树，或者一个叶子节点而言，它的ML（最大路径）是 0, 深度是 1。

简单二叉树

假设我们定义一棵二叉树树，它只有两个主要分枝，其他都是细小的分枝（长度为 1），那么我们称其为简单二叉树，则容易证明

对于一个深度只有2的简单二叉树而言，我们容易知道它的ML = 2。

复杂二叉树

如果一个树非常的复杂，其不满足简单二叉树的定义，换言之我们无法保证式（1）成立，但我们都可以想到，即使在复杂的二叉树都可以分解为多个简单二叉树。换言之，我们将复杂二叉树假想为一个通信底层中的分治系统，而简单二叉树可以看出其中的子系统。
在分治系统中，我们知道每个子系统需要向其它子系统分享其路由信息，但为了保证安全，并不会暴露自身的系统结构。但是我们仍然可以用某个量从上方去比较每个子系统。

即，假设定义一个全局变量来表示 ML（比较量），每进入一个子系统（简单二叉树）中，都将其内部的 ML 与全局的 ML 进行比较。子系统的 ML可有式（1）得到。子节点（子系统）向其父节点（这个就是其他系统）分享的是它的深度。因为其父节点对它的ML不感兴趣，只对它的深度感兴趣。当遍历所有的子系统后，自然就能得到整个系统的 ML。

总结

  1. 利用全局变量来解决类似于”任意xxx”的问题。
  2. 递归思想，从简单到复杂。
  3. 利用基本方法来解决复杂问题。
